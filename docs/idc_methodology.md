# Imperial Development Cycle (IDC) - Standard Operating Procedure

To ensure the highest engineering standards and prevent technical debt, the OmniSync Hub project follows the **IDC (Imperial Development Cycle)**. This "Blueprint-First" methodology is inspired by the rigorous processes of Big Tech corporations.

## 🛡️ The 4-Step Mandatory Workflow

### Phase 1: Blueprint (PRD & OpenSpec)
*   **Owner:** Agent 27 (Imperial Architect).
*   **Outcome:** `PRD.md` (Product Requirements Document).
*   **Focus:** Business value, user stories, and acceptance criteria.

### Phase 2: Engineering (Technical Design)
*   **Owner:** Agent 27 & Core Architect.
*   **Outcome:** `TECH_DESIGN.md`.
*   **Focus:** Data contracts (Protobuf), Schema design, and Backend Trio logic mapping.

### Phase 3: Security Gate (Audit & Compliance)
*   **Owner:** Agent 08 (Regulatory) & Agent 25 (Blue Team).
*   **Outcome:** `SECURITY_AUDIT.md`.
*   **Focus:** Threat modeling, Pentagon Layer isolation, and regulatory alignment.

### Phase 4: Execution (Atomic Implementation)
*   **Owner:** Domain-specific Expert Agent.
*   **Outcome:** Production-ready source code.
*   **Standard:** All tasks must be broken down into **Atomic Tasks (< 100 LOC)** to ensure AI accuracy and context window safety.
